# Key Takeaways

## Overview

This lab focused on implementing IT security policies through technical controls. The main lesson was that policies must be backed by enforceable settings, validation, and documentation.

## Main Takeaways

### 1. Policy needs enforcement

A written policy is important, but it is not enough by itself. Group Policy, endpoint protection settings, baselines, and mobile controls help make policy enforceable.

### 2. Group Policy supports centralized management

Group Policy allows administrators to configure settings once and apply them across domain users and computers.

### 3. Password policy should be measurable

A password policy can define requirements such as history, maximum age, minimum length, and complexity.

### 4. Password validation matters

Testing both failed and successful password changes confirms that the policy is working as intended.

### 5. Antivirus settings should not depend on users

Users should not be able to disable critical endpoint protections. Central management helps keep real-time protection active.

### 6. Security baselines save time

Microsoft security baselines provide recommended configuration sets that administrators can review, import, customize, and deploy.

### 7. Account lockout policies reduce brute-force risk

Lockout thresholds and reset timers help slow down automated password guessing attacks.

### 8. Mobile devices need security controls too

Mobile security should include malware scanning, updates, lock screens, encryption, and recovery/location features.

### 9. Acceptable use and privacy policies support governance

Acceptable use policies tell users what behavior is allowed. Privacy policies explain how personal data is collected, used, shared, stored, and protected.

## Reflection

This lab showed how security governance connects to technical implementation. Policies are strongest when they are clear, centrally enforced, tested, and reviewed regularly.
